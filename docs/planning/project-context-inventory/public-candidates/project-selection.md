# Portfolio Public Candidate Selection

Status: draft
Last checked: 2026-06-28
Purpose: decide which dossier material can become portfolio layout content

## Selection Rule

Public pages should not mirror full dossiers. Each project should expose only:

- one-line project thesis
- why it started
- 2-3 important choices
- how the user used AI
- current state and why
- one lesson or method gained

Raw diary, raw Claude/Codex logs, internal paths, credentials, and unresolved
verification details stay in dossiers.

## Layout Implication

Recommended first portfolio layout:

- Hero/intro: "AI로 프로젝트를 많이 만들었다"가 아니라 "AI와 함께 시작, 보류, 폐기, 검증 기준을 남겼다."
- Main case grid: 6-9 large cards from `Main Cases`.
- Archive strip: 3-5 stopped projects with "why stopped" as the headline.
- Tool shelf: smaller secondary cards for personal/operational tools.
- No raw evidence on public pages. Link each public card back to an internal dossier only in private review mode.
